## 架构红线（不可违反）

> 以下两条是"防止架构退化"的保险丝

### 红线 1：Interaction Orchestrator 拥有 UI 最终裁决权

```
AI（任何角色）永远不能直接决定 UI 的呈现方式，只能提出请求。
```

### 红线 2：预判指令默认需人确认

```
任何 Predicted Action，默认必须是"需人确认"的。
绝对禁止：AI 看你可能要做 → 就帮你做了。
```
